# v0.1 实现计划：飞书 AI 助手 Bot 基础能力

## 目标

让飞书 Bot 作为 AI 对话助手跑通并稳定运行。这是所有后续功能的基础。

**一句话：** 用户在飞书私聊 Bot，Bot 用 Claude 回复，支持多轮对话。

---

## 用户故事

> 作为团队成员，我可以在飞书里找到这个 Bot，私聊它问任何问题，它会像 Claude 一样回复我，并且记住我们这次对话的上下文。

---

## 范围

### 做

- 飞书私聊收发消息
- Claude 多轮对话（记住同一会话的上下文）
- 回复 JSON 交互卡片（有样式，不是纯文字）
- 收到消息先 emoji 反应（"正在输入..."），再回复卡片
- 部署到 Render，稳定运行

### 不做（留给后续版本）

- 群聊 / 话题群（v0.2）
- 妙记总结（v0.2）
- 客户信息查询（v0.3）
- CRM 同步（v0.3）
- 权限系统（v0.4）

---

## 技术实现

### 现有能力（已完成）

- ✅ FastAPI webhook 接收飞书消息
- ✅ Claude Messages API 调用
- ✅ 飞书卡片回复
- ✅ Render 部署

### 需要完善

- [ ] emoji 反应使用飞书官方推荐表情
- [ ] 对话历史重启后丢失 → 用 SQLite 持久化（简单 KV：thread_id → messages JSON）
- [ ] 去掉 `ingest_service`（v0.1 不需要，减少启动复杂度）
- [ ] `AGENT_ID` 不再是必填项（v0.1 用 Messages API，不用 Managed Agents）

---

## 成功标准

- [ ] 飞书私聊 Bot，Bot 在 10 秒内用卡片回复
- [ ] 连续发 3 条消息，Bot 记住上下文
- [ ] 重启服务后，同一私聊继续对话，Bot 仍记得之前说了什么
- [ ] Render 上稳定运行，不因为 ingest_service 报错崩溃

---
